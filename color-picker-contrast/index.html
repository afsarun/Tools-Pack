Color Accessibility Best Practices

🎨 Don't Rely on Color Alone
Use additional indicators like patterns, text labels, or icons to convey meaning beyond color differences.
👁️ Test with Color Blindness Simulators
Check how your colors appear to users with various types of color vision deficiency.
📱 Consider Different Lighting Conditions
Test your colors in bright sunlight, dim lighting, and various screen brightness levels.
🔄 Provide Alternative Contrast Options
Offer high contrast modes or allow users to customize color schemes.

WCAG 2.1 Compliance Levels

Level AA (Minimum)

  • Normal text: ≥4.5:1 contrast ratio
  • Large text (18pt+): ≥3:1 contrast ratio
  • UI components: ≥3:1 contrast ratio

Required for most accessibility regulations.

Level AAA (Enhanced)

  • Normal text: ≥7:1 contrast ratio
  • Large text (18pt+): ≥4.5:1 contrast ratio

Recommended for highest accessibility.

What counts as "large text"?

  • 18 points regular weight (≈ 24px)
  • 14 points bold weight (≈ 18.67px)
  • Icons and graphical objects require 3:1 contrast

How to Use

  1. Pick a Text and Background color using the color inputs or paste a hex value (e.g., #1a73e8 or #abc).
  2. The tool instantly shows the contrast ratio and pass/fail badges for AA (normal/large) and AAA (normal).
  3. Preview how the pair looks on Normal and Large text. "Large" means ≥18pt normal or ≥14pt bold.
  4. Click Copy to copy the hex for either color.
  5. All calculations run locally—nothing leaves your browser.

Tips

  • Only solid colors are supported. For semi-transparent overlays, first compute the flattened color against the background.
  • #RGB is auto-expanded to #RRGGBB.
  • For non-text UI (icons, borders), WCAG suggests ≥3:1 against the adjacent color.
