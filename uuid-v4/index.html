📏 Keep a consistent format
Pick lowercase with hyphens for readability, or hyphenless for compact storage — and stick to it.
⚙️ Indexing considerations
Random keys can fragment B-trees; consider UUIDv7 for time-ordered needs, or store as 16-byte binary.

Generate UUID v4 with crypto.getRandomValues for entropy.
Security note: Avoid Math.random() or time-based hacks.

All UUIDs are generated locally using the browser’s cryptographic PRNG.

📜 RFC 4122 specifics

  • Version: 4 (random) → the high nibble of byte 6 is 0100
  • Variant: RFC 4122 → the two high bits of byte 8 are 10
  • Layout: xxxxxxxx-xxxx-4xxx-yxxx-xxxxxxxxxxxx where y is 8–b

📋 When to use

  • Database keys, request IDs, idempotency keys
  • Client-side identifiers that don’t reveal sequence/time
  • Cross-system correlation without central coordination

🔑 Uniqueness

  • UUID v4 has 122 random bits (after version/variant) → astronomically low collision probability for practical use.
  • Still treat UUIDs as identifiers, not secrets.

⚙️ How to use

💡 Tips

  • Prefer lowercase for consistency in logs and URLs.
  • Use hyphenless for compact storage or slug-like IDs (still validate as UUID format if required).
  • Never expose UUIDs as authentication secrets — pair with access control.
