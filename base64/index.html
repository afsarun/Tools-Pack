Base64 Basics

What this tool can do

  • Encode text or files to Base64
  • Decode Base64 back to text or files
  • URL-safe Base64 (Base64URL) conversion
  • Data URLs for images and other assets

All processing happens locally in your browser; nothing is uploaded.

Inputs

  • Text: UTF-8 safe (emoji and non-Latin supported)
  • Files: drag & drop or file picker (images, PDFs, etc.)
  • Options: URL-safe variant, strip/add padding, line wrap

Standards & variants

  • Standard Base64: alphabet A–Z a–z 0–9 + /, padding =
  • Base64URL: replace + → -, / → _; padding often omitted
  • MIME wrap: optional line breaks every 76 chars; whitespace is ignored by decoders

What counts as “valid Base64”?

  • Characters from the selected alphabet (standard or URL-safe)
  • Optional = padding (0–2 chars) to a length divisible by 4
  • Whitespace/newlines are allowed and ignored by robust decoders

How to use

  1. Paste text or drop a file to Encode to Base64 (or select Decode for the reverse).
  2. Toggle URL-safe, Padding, or Line wrap options as needed.
  3. Copy the result, or for files use Download to save the decoded blob.
  4. For images, select Data URL to prepend data:<mime>;base64, automatically.

Tips

  • If your Base64 lacks padding, enable “auto-pad” or add = to reach length % 4 == 0.
  • “URL-safe” mode is required for JWTs, signed URLs, and some APIs.
  • Large files use memory; your browser limits practical size (hundreds of MB may be slow).

Base64 Best Practices

🔤 Use UTF-8 for text

Always encode/decode strings via TextEncoder/TextDecoder (avoids atob/btoa Unicode issues).

🔁 Strip or add padding as required

APIs differ: standard Base64 expects =, while Base64URL often removes it.

🧹 Ignore whitespace

Robust decoders should remove spaces/newlines before processing.

📏 Prefer small embeds

Data URLs bloat HTML/CSS if large—inline only tiny assets (e.g., <10–20 KB).
