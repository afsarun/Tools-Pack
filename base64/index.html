Base64 Best Practices

🔤 Use UTF-8 for text
Always encode/decode strings via TextEncoder/TextDecoder (avoids atob/btoa Unicode issues).
🔁 Strip or add padding as required
APIs differ: standard Base64 expects =, while Base64URL often removes it.
🧹 Ignore whitespace
Robust decoders should remove spaces/newlines before processing.
📏 Prefer small embeds
Data URLs bloat HTML/CSS if large — inline only tiny assets (e.g., <10–20 KB).

Technical Implementation

  • Options: URL-safe variant, strip/add padding, line wrap

📑 Standards & Variants

  • Standard Base64: alphabet A–Z a–z 0–9 + /, padding =
  • Base64URL: replace + → -, / → _; padding often omitted
  • MIME wrap: optional line breaks every 76 chars; whitespace ignored

🎯 What counts as “valid Base64”?

  • Characters from the selected alphabet (standard or URL-safe)
  • Optional = padding (0–2 chars) to make length divisible by 4
  • Whitespace/newlines allowed and ignored by robust decoders

🧩 How to use

  • Paste text or drop a file to Encode (or select Decode for reverse)
  • Toggle URL-safe, Padding, or Line wrap options
  • Copy the result, or for files use Download to save the decoded blob
  • For images, select Data URL to prepend data:<mime>;base64

💡 Tips

  • If Base64 lacks padding, enable “auto-pad” or add = until length % 4 == 0
  • “URL-safe” mode is required for JWTs, signed URLs, and some APIs
  • Large files use memory; browsers may slow with hundreds of MB
